# 接入指南（JitPack · 仅依赖）

## 1. 模块说明（你需要依赖谁）

| 模块 | 作用 | 是否必选 |
|------|------|----------|
| **smart** | 通用 RecyclerView 适配封装（列表、多类型、侧滑、选择等） | **必选** |
| **node** | 在 smart 之上的树形 / 多级列表（展开折叠、节点增删改） | 按需 |
| **app** | 本仓库自带 Demo，**不要** `implementation` 到你的工程里 | — |

只用普通列表：只加 **smart**。要做树形列表：加 **smart + node**（node 会依赖 smart 的能力，请同时保持 **同一 Tag**）。

---

## 2. 添加 JitPack 仓库

### 方式 A：Gradle 7+（`settings.gradle` / `settings.gradle.kts` 推荐写法）

在 `dependencyResolutionManagement { repositories { ... } }` 中增加：

**Groovy**

```groovy
maven { url 'https://jitpack.io' }
```

**Kotlin DSL**

```kotlin
maven { url = uri("https://jitpack.io") }
```

本仓库根工程示例见 `settings.gradle` 中的 `google()`、`mavenCentral()` 与 `jitpack.io` 并列配置。

### 方式 B：仍使用根目录 `build.gradle` 的 `buildscript` / `allprojects`

在 `repositories` 中加入同上 `maven { url 'https://jitpack.io' }` 即可。

## 3. 添加依赖（坐标以 JitPack 页面为准）

1. 打开 [JitPack - XAdapter](https://jitpack.io/#xiaohaozi9825/XAdapter)  
2. 选择一个 **已构建成功的 Git 提交或 Tag**（页面会显示绿色 `Get it`）  
3. 点击 **Gradle**，复制页面生成的依赖行；或按下文格式自行替换 **`<Tag>`**。

**Groovy（`app` 模块 `dependencies`）**

```groovy
dependencies {
    // 必选：核心 SDK
    implementation 'com.github.xiaohaozi9825:XAdapter:smart:<Tag>'
    // 可选：树形 / 多级列表
    implementation 'com.github.xiaohaozi9825:XAdapter:node:<Tag>'
}
```

**Kotlin DSL**

```kotlin
dependencies {
    implementation("com.github.xiaohaozi9825:XAdapter:smart:<Tag>")
    implementation("com.github.xiaohaozi9825:XAdapter:node:<Tag>")
}
```

说明：

- **`<Tag>`** 与你在 GitHub 打的 **Tag 名** 或 **commit hash** 一致即可（例如 `2.1.0`）。  
- 子模块坐标格式为：`com.github.<GitHub用户名>:<仓库名>:<子模块名>:<版本>`。  
- 若 JitPack 页面展示的字符串与上文略有差异，**以页面一键复制为准**。

## 4. 工程侧必开能力：ViewBinding

本库列表项基于 **ViewBinding**（`ViewBinding` 泛型）。请在 **使用到 XAdapter 的模块**（一般是 `app`）开启：

**Groovy**

```groovy
android {
    buildFeatures {
        viewBinding true
    }
}
```

若你同时使用 DataBinding，可与本库并存，按需同时打开 `dataBinding true`（与 Demo `app` 模块一致即可）。

## 6. 常见问题

**Q：Sync 报找不到 `com.github.xiaohaozi9825:XAdapter:...`？**  
先确认已加 **JitPack** 仓库，且 **能访问外网**；再在 JitPack 网页上对该 Tag 点一次 **Get it**，等构建成功后再依赖。

**Q：只想要树形列表，可以只引 node 吗？**  
不建议。请 **smart + node** 同版本，或严格按 JitPack 对 `node` 产物声明的传递依赖为准。

**Q：和 Demo 里代码怎么对应？**  
Demo 在模块 **`app`** 下，包名 `pw.xiaohaozi.xadapter`；可参考其中 `fragment` 命名与 `smart` / `node` 的用法，**不必**依赖 `app` 模块。
