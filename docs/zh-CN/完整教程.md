## 1. 核心概念

| 名词 | 含义 |
|------|------|
| **smart** | SDK 主模块：`SmartAdapter` 继承 `XAdapter`，组合「数据 + 事件 + 选中」等能力。 |
| **node** | 在 smart 之上：树形数据扁平化后的 `NodeAdapter`。 |
| **Provider** | 一种「布局 + 数据类型」的绑定单元；多布局时会有多个 `SmartProvider` / `NodeProvider`。 |
| **`OnBindParams`** | 每次 item 绑定时的参数包：`holder`、`data`、`position`、`payloads`，且 **本身是 `CoroutineScope`**，可用 `launch` 做与本次绑定对齐的异步。 |

**`position` 说明**：存在头布局等时，`position` 多为 **Adapter 位置**。取数据列表下标请用 `adapter.getDataPosition(position)`（详见 [API 参考 - XAdapter](./API参考-smart.md) 第六节）。

## 2. RecyclerView 与数据

```kotlin
recyclerView.adapter = adapter
// 建议同时设置 LayoutManager
recyclerView.layoutManager = LinearLayoutManager(context)
```

首次展示前需有数据，例如：

```kotlin
adapter.refresh(list)       // 保留内部 list 引用并整体替换内容（非 Differ）
// 或
adapter.setList(mutableList) // 替换整个 MutableList 引用（非 Differ）
```

具体 API 含义见 [§9](#9-数据增删改与-differ) 与 [API 参考 - 数据](./API参考-smart.md#22-smartdataproxy)。

## 3. smart：单布局列表

使用扩展函数 **`createAdapter`**（包：`pw.xiaohaozi.xadapter.smart.ext`）。

绑定回调类型为 **`SmartAdapter<VB, D>.(OnBindParams<VB, D>) -> Unit`**：最后一个 lambda 即 `bind`，**参数是 `OnBindParams`**，不要写成 `(holder, data)` 两个参数。

```kotlin
import pw.xiaohaozi.xadapter.smart.ext.createAdapter

val adapter = createAdapter<ItemVerseBinding, VerseInfo> { p ->
    p.binding.tvContent.text = p.data.content
    p.binding.tvAuthor.text = p.data.author
}
```

可选参数（按需）：

- **`itemType`**：显式 itemType，默认 `0`。
- **`onItemId`**：稳定 ID，用于 `RecyclerView` 局部动画等。
- **`init`**：在 `SmartProvider` 上初始化（如链式 `setOnClickListener`）。
- **`created`**：`ViewHolder` 创建后只执行一次的初始化。

## 4. smart：多布局列表

1. 使用 **`createAdapter()`**（无泛型实参版本）得到 **`SmartAdapter<ViewBinding, Any?>`**，以容纳多种数据类型；也可使用 **`customItemType`** 自定义类型映射（见 Demo）。
2. 在 **`SmartAdapter` 或 `SmartProvider`** 上链式调用 **`withType<子VB, 子D> { ... }`** 注册多种类型（`子D` 可为不同数据类）。
3. 最后调用 **`toAdapter()`** 回到 `SmartAdapter` 继续配置或赋值给 `RecyclerView`。

```kotlin
import pw.xiaohaozi.xadapter.smart.ext.createAdapter

val adapter = createAdapter()
    .withType<ItemVerseBinding, VerseInfo> { p ->
        p.binding.tvContent.text = p.data.content
    }
    .withType<ItemImageCardBinding, ImageItem> { p ->
        p.binding.image.setImageResource(p.data.resId)
    }
    .toAdapter()
```

若多种数据有 **共同父类型**（如 `sealed class Row`），也可使用 **`createTypeAdapter<ViewBinding, Row>()`**，此时 `withType` 的 `子D` 均为 `Row` 的子类。

## 5. 生命周期与协程

在 `Activity` / `Fragment` 上：

```kotlin
adapter.bindLifecycle(this)
```

作用：Adapter 与页面生命周期对齐，并在内部取消协程等（见 `XAdapter` 实现）。

**在 `bind` / `withType` 里发协程**：优先使用 **`OnBindParams` 作为 receiver**（`p.launch { }`），这样 `ViewHolder` 再次绑定或回收时，与 **本次绑定** 对齐的子协程会被取消；需要 Adapter 级长生命周期任务时再使用 Adapter 自身 `CoroutineScope`。

## 6. 特殊布局：头 / 脚 / 空 / 缺省页

均在 **`SmartAdapter` / `XAdapter` 扩展** 上配置（链式返回 `this`）：

| DSL / 方法 | 作用 |
|-------------|------|
| `addHeader { }` | 头布局 |
| `addFooter { }` | 脚布局 |
| `setEmpty { }` | 无数据时的空布局 |
| `setDefaultPage { }` | 缺省页（网络错误等），配合 `showDefaultPage` / `hintDefaultPage` |

分组场景下子项可 **`isFixedViewType()` 占满一行**（`GridLayoutManager` / `StaggeredGridLayoutManager` 下跨列）。

## 7. 事件监听

在 **`SmartAdapter`** 或 **`SmartProvider`** 上调用（`SmartAdapter` 与 `SmartProvider` 均实现 `EventProxy`）：

| API | 说明 |
|-----|------|
| `setOnClickListener(id = R.id.xx) { }` | `id` 为空则整项点击 |
| `setOnLongClickListener` | 返回 `Boolean` 表示是否消费 |
| `setOnCheckedChangeListener` | `CompoundButton` |
| `setOnTextChange` | `TextView` / `EditText` 文本变化 |

扩展 **`onClick` / `onLongClick`** 可把 lambda 简化为直接收到 `binding`（见 `SmartAdapterExt`）。

## 9. 数据增删改与 Differ

**普通模式**（默认）：通过 **`SmartDataProxy`** 暴露的方法操作 **`getDataList()`** 对应的列表，例如 `add`、`removeAt`、`update`、`swap`、`refresh`、`setList` 等。

**注意**：进入 **Differ** 模式后，**不得**再使用上述多数就地修改 API，应使用 **`submitList`**。

开启 Differ：

```kotlin
adapter.setDiffer(DiffUtil.ItemCallback<YourData>()) // 或传入 AsyncDifferConfig
adapter.submitList(newList)
```

## 10. 侧滑、拖拽

在 **`SmartAdapter`** 或 **`SmartProvider`** 上链式调用扩展：

| 扩展 | 作用 |
|------|------|
| `swipeMenu()` | 侧滑菜单（item 根布局需为库内 `SwipeItemLayout`） |
| `swipeDelete { }` | 侧滑删除 |
| `dragSort { }` | 拖拽排序 |

具体参数含义见源码注释与 Demo。

## 11. node：树形列表

1. 数据类型实现 **`NodeEntity<Parent, Child>`**；需要展开折叠的实现 **`ExpandedNodeEntity`**。
2. 使用 **`nodeAdapter`**（`pw.xiaohaozi.xadapter.node.ext`）创建 **`NodeAdapter`**，用法与 smart 的 `createAdapter` 类似，**`bind` 同样接收 `OnBindParams`**。
3. 维护 **`adapter.source`** 树形根列表，调用 **`adapter.refresh(list)`** 或 **`refresh()`** 同步到扁平列表。
4. **`expand` / `collapse`** 可按位置或全局控制子树展示。

多布局：`nodeAdapter()` 无泛型推断版本 + **`NodeAdapter.withType`** + **`NodeProvider.toAdapter()`**（见 [API 参考 - node](./API参考-node.md)）。
