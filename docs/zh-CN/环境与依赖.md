# 环境与依赖版本

本文说明：**你的 App 工程** 与 **XAdapter SDK（smart / node）** 之间，在版本上如何对齐。数值以本仓库当前源码为准；发版后若升级，以对应 **Git Tag** 下的 `build.gradle` 为准。

---

## 1. SDK 自身编译参数（smart / node）

| 项目 | 值 |
|------|-----|
| compileSdk | 33 |
| minSdk | 21 |
| targetSdk（库 `defaultConfig`） | 33 |
| Java / Kotlin JVM | 1.8（`jvmTarget = '1.8'`） |

含义简述：

- **minSdk 21**：你的 `application` 若 `minSdk` 低于 21，则不能降到 21 以下仍使用本库。  
- **compileSdk**：建议你的工程 **compileSdk ≥ 33**，避免与新 API / 依赖不一致。

---

## 2. 本仓库构建 SDK 时使用的插件版本（参考）

根目录 `build.gradle` 中：

| 插件 | 版本 |
|------|------|
| Android Gradle Plugin (AGP) | 7.3.0 |
| Kotlin Android | 1.7.10 |

说明：JitPack 会使用你 **Tag 指向的提交** 里的 Gradle 配置来编 AAR。  
你的工程 **不必** 与上述版本完全一致，但建议：

- AGP **不要过旧**（例如长期停留在 4.x），以免无法解析较新的 AndroidX 依赖。  
- Kotlin 版本与工程其余部分统一即可；若编译报 Kotlin 元数据不兼容，再考虑升级 Kotlin。

---

## 3. SDK（smart）传递到你工程里的主要依赖

`smart/build.gradle` 中显式声明（随版本可能微调）：

| 依赖 | 版本（当前） |
|------|----------------|
| androidx.lifecycle:lifecycle-runtime-ktx | 2.4.0 |
| androidx.core:core-ktx | 1.7.0 |
| com.google.android.material:material | 1.7.0 |

**RecyclerView**：当前 `smart` 未单独写死 `recyclerview` 依赖，一般通过 **Material** 等传递引入。为稳妥，建议在 **app 模块** 显式增加一行（版本可按你工程统一矩阵选择，例如 1.3.x）：

```groovy
implementation 'androidx.recyclerview:recyclerview:1.3.2'
```

这样不依赖「Material 是否一定带上 RecyclerView」的细节。

**node 模块**：在以上基础上，同样依赖 `lifecycle-runtime-ktx`、`core-ktx`、`material`；并与 **smart** 源码级关联，发版时请 **smart 与 node 使用同一 Tag**。

---

## 4. 可选：kotlin-reflect

`smart` 内个别分支在 **数据为 null 且需推断 itemType** 时会尝试使用 Kotlin 反射；若未添加 `kotlin-reflect`，会捕获异常并走降级逻辑（见源码注释）。

一般 **不必** 主动加。若你希望该分支行为更稳定，可自行添加与工程 Kotlin 主版本一致的反射库，例如：

```groovy
implementation "org.jetbrains.kotlin:kotlin-reflect:1.7.10"
```

（版本号请与工程 `kotlin_version` 对齐。）

---

## 5. 与 Demo（app）的关系

`app` 模块中还有 **演示用** 第三方库（下拉刷新、图片、权限等），**不会**随 `smart` / `node` 的 AAR 进入你的工程。  
你只需关心本文 **§3** 中与列表相关的 AndroidX 依赖即可。
